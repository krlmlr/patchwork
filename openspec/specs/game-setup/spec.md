### Requirement: GameSetup encodes the initial patch circle arrangement

`GameSetup` SHALL hold the initial circular arrangement of all 33 patches as a `std::array<uint8_t, 33>` and the `uint64_t` seed used to generate it.

#### Scenario: from_seed produces a valid permutation

- **WHEN** `GameSetup::from_seed(s)` is called for any seed `s`
- **THEN** the resulting circle contains each of the values `0` through `32` exactly once

#### Scenario: from_seed is deterministic

- **WHEN** `GameSetup::from_seed(s)` is called twice with the same seed `s`
- **THEN** both results have identical circle arrays

#### Scenario: from_seed produces distinct permutations for different seeds

- **WHEN** `GameSetup::from_seed(s1)` and `GameSetup::from_seed(s2)` are called with `s1 ≠ s2`
- **THEN** the two circle arrays are not equal (with overwhelming probability)

### Requirement: GameSetup can be loaded from a YAML setup file

A `GameSetup` SHALL be constructible from a YAML file produced by `codegen/generate_setups.R`. The loader SHALL recover the seed and circle array exactly.

#### Scenario: Load round-trip

- **GIVEN** a YAML setup file written by the R generator (e.g. `data/setups/setup-00001.yaml`)
- **WHEN** `GameSetup::load(path)` is called with that file
- **THEN** the returned `GameSetup` has the same `seed` value and the same `circle` array as recorded in the file
- **AND** the circle is a valid permutation of `[0 … 32]`

#### Scenario: Load rejects malformed input

- **WHEN** `GameSetup::load(path)` is called with a file missing the `circle:` line
- **THEN** a `std::runtime_error` is thrown

### Requirement: GameSetup serialises to an NDJSON record

`GameSetup::to_ndjson` SHALL emit exactly one JSON line to the provided `std::ostream` containing the type tag, seed, and circle array.

#### Scenario: NDJSON output is a single line with correct fields

- **WHEN** `to_ndjson(out)` is called on a `GameSetup`
- **THEN** the output contains a single newline-terminated JSON object
- **AND** the object has `"type": "setup"`, `"seed": <value>`, and `"circle": [<33 ids>]`

### Requirement: Canonical setups are versioned in `data/setups/`

The repository SHALL contain at least 10 committed YAML setup files generated by `codegen/generate_setups.R`, named `setup-00001.yaml` through `setup-00010.yaml`.

#### Scenario: Canonical setup files are present and valid

- **WHEN** the repository is cloned
- **THEN** `data/setups/setup-00001.yaml` through `data/setups/setup-00010.yaml` exist
- **AND** each file contains a valid 33-element circle array and a matching seed

### Requirement: GameSetup is unit tested

All `GameSetup` behaviours SHALL have Catch2 unit tests covering `from_seed`, `load`, and `to_ndjson`.

#### Scenario: Tests exist and pass

- **WHEN** `meson test -C build` is run
- **THEN** all game-setup tests pass with exit code 0
